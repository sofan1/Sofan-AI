# How to Push This Workspace to GitHub + Publish

You have a clean, sanitized git repo ready to push. The sandbox has no
GitHub credentials and we don't accept PATs in chat (that's how the
previous keys leaked). Push from your own machine.

## Step 1 — Download the workspace

Download this file from the sandbox:

  /home/z/my-project/download/sofan-ai-hermes-workspace.tar.gz

(Size: ~31 KB. Contains only source — no secrets, no node_modules,
no .next build cache. Verified clean.)

## Step 2 — Extract on your machine

```bash
mkdir sofan-ai-hermes-workspace
cd sofan-ai-hermes-workspace
tar -xzf /path/to/sofan-ai-hermes-workspace.tar.gz --strip-components=1
```

## Step 3 — Create a new GitHub repo

Go to https://github.com/new and create a new repo:
- Name: `sofan-ai-hermes`  (or whatever you prefer)
- Visibility: **Private** (recommended — contains deploy config)
- Do NOT initialize with README/.gitignore (already included)

## Step 4 — Push from your machine

```bash
cd sofan-ai-hermes-workspace
git remote add origin git@github.com:sofan1/sofan-ai-hermes.git
git push -u origin main
```

(Use HTTPS with a PAT stored in a credential manager if you don't
have SSH set up. NEVER paste PATs in chat.)

## Step 5 — Publish on the platform

The platform deploys THIS workspace directly (it runs `npm start`
on port 3000). You have two paths:

### Option A — Publish the current sandbox (recommended)

The sandbox is already publish-ready:
- ✓ Production build present (`.next/BUILD_ID`)
- ✓ `npm start` script defined
- ✓ All services auto-spawn on first request
- ✓ No external dependencies

Just click **Publish** in the platform UI. The platform will run
`npm install && npm run build && npm start` and route the preview
URL to port 3000.

### Option B — Deploy from the GitHub repo (for a fresh environment)

If you want to deploy from the GitHub repo instead (e.g. on a different
platform or your own VPS):

```bash
git clone git@github.com:sofan1/sofan-ai-hermes.git
cd sofan-ai-hermes
npm install
npm run build
npm start
# Now serving on http://0.0.0.0:3000
```

For a VPS with systemd, use `scripts/start_all.sh` (it spawns all
three services detached).
