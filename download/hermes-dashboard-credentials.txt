DASHBOARD ACCESS (NO AUTH — TEMPORARY)
------------------------------------------------------------------
You asked to remove auth so you can configure everything from the UI.
The Hermes dashboard is bound to 127.0.0.1 (loopback) — Hermes skips
the basic_auth gate when on loopback. The Next.js proxy forwards
requests to it, so the published URL gives you auth-free access.

WHEN YOU'RE READY TO RE-ENABLE AUTH:
  1. Edit /home/z/.hermes/config.yaml
  2. Add a basic_auth block:
       dashboard:
         bind: 0.0.0.0          # change from 127.0.0.1
         port: 9119
         basic_auth:
           enabled: true
           username: <your-username>
           password_hash: "<bcrypt-hash>"
  3. Generate a bcrypt hash with:
       cd /home/z/.hermes/hermes-agent
       .venv/bin/python -c "from plugins.dashboard_auth.basic import hash_password; print(hash_password('<your-password>'))"
  4. Restart Hermes:
       pkill -f "hermes dashboard"; /home/z/my-project/scripts/start_all.sh
